# Agents

Subagent definitions. Each file here defines one agent persona that the Conductor can dispatch.

**Coming in Phase 1:**
- `conductor.md` — top-level orchestrator
- `onboarding.md` — first-run org interview
- `governance-validator.md` — checks drafts against governance rules
- `sop-author.md` — renders human-facing Markdown
- `qa-evaluator.md` — scores SOPs against rubric
- `pillar-acquisition.md` (and one per pillar) — domain specialists
- `org-mapper.md` — assigns RACI, runs gap analysis
- `tool-mapper.md` — matches steps to inventory

See [`../docs/architecture.md`](../docs/architecture.md) for the full topology.
